# 📜 История реализации проекта COZY ART

## Общая информация
- **Проект:** Многостраничный сайт с CMS и SEO для "COZY ART — Архитектурный бетон"
- **Старт проекта:** 2025-01-XX
- **Статус:** Инициализация
- **Стек:** Next.js 14, TypeScript, Tailwind CSS, GSAP, Sanity.io, Docker, Nginx

---

## 🎯 Цели проекта
1. Создать полноценный многостраничный сайт на основе HTML-шаблона
2. Интегрировать headless CMS для управления контентом
3. Реализовать полную SEO-оптимизацию
4. Подготовить Docker-контейнер для развертывания единым стеком

## 🗂️ Структура проекта (актуальная)

```
/workspace
├── app/                          # Next.js App Router
│   ├── (main)/                   # Основная группа роутов
│   │   ├── page.tsx              # Главная
│   │   ├── catalog/              # Каталог
│   │   ├── products/[slug]/      # Товар
│   │   ├── projects/             # Проекты
│   │   ├── about/                # О компании
│   │   ├── process/              # Процесс
│   │   ├── blog/                 # Блог
│   │   └── contacts/             # Контакты
│   ├── api/                      # API роуты
│   ├── layout.tsx                # Корневой layout
│   └── globals.css               # Глобальные стили
├── components/                   # React компоненты
│   ├── ui/                       # Базовые UI компоненты
│   ├── layout/                   # Header, Footer
│   └── sections/                 # Секции страниц
├── lib/                          # Утилиты и конфиги
│   ├── sanity.ts                 # Sanity клиент
│   └── utils.ts                  # Хелперы
├── public/                       # Статические файлы
├── sanity/                       # Sanity схемы
├── docker-compose.yml            # Docker конфигурация
├── Dockerfile                    # Docker образ
├── next.config.js                # Next.js конфиг
├── tailwind.config.js            # Tailwind конфиг
├── tsconfig.json                 # TypeScript конфиг
└── README.md                     # Документация
```

## 📝 Заметки и решения

### Принятые архитектурные решения
- **App Router** вместо Pages Router для лучшей производительности
- **Server Components** по умолчанию для уменьшения bundle size
- **ISR** для баланса между статикой и актуальностью данных
- **Sanity.io** как наиболее гибкая headless CMS для данного проекта

### Потенциальные риски
- Сложность миграции анимаций GSAP на React компоненты
- Необходимость оптимизации большого количества изображений
- Настройка правильной стратегии кэширования
